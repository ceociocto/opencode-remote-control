## Affected Pages/Routes

This is a **backend bot service** — no web pages. Test via:
- Telegram chat with the bot
- Mock Telegram API for automated tests

## Key Interactions to Verify

### Telegram Bot
- **Send prompt** — User sends "Add a README file" → bot responds with OpenCode output
- **Reply in thread** — User replies in same thread → session context maintained
- **New thread** — User starts new conversation → new session created
- **Command /approve** — During pending file edit → `/approve` allows edit
- **Command /reject** — During pending file edit → `/reject` denies edit
- **Command /diff** — Shows diff of pending changes in code block
- **Command /files** — Lists files modified in current session

## Edge Cases

- **Long message (>4096 chars)** — Should split into multiple messages
- **Concurrent messages in same thread** — Should queue and process sequentially
- **Approval timeout (5 min)** — Should auto-reject if user doesn't respond
- **Session timeout (30 min idle)** — Session should be cleaned up
- **OpenCode not running** — Should return "OpenCode is offline" message
- **Telegram rate limit hit** — Should queue messages and respect `retry_after`

## Critical Paths

1. **Happy path:** Send prompt → OpenCode processes → response streams back
2. **Approval path:** OpenCode requests file edit → bot asks for approval → user approves → edit proceeds
3. **Rejection path:** OpenCode requests file edit → bot asks for approval → user rejects → edit denied
4. **Error path:** OpenCode crashes → bot notifies user of failure
